# BERT vs Ollama 实体抽取能力对比测试报告

## 测试时间
2026-04-08

## 测试目的
对比BERT和Ollama (qwen25chat) 在法律案件实体抽取任务上的能力差异，评估是否可以用BERT替代Ollama来提升速度。

## 测试案例
```
公诉机关指控：一、××2016年5月29日19时许，被告人杨1某伙同杨某2、李某1（均已判刑）经事先预谋，
以戴帽子、口罩、手套等方式进行伪装后，敲门进入杨某1居住的霞浦县牙城镇中心小学旁造福工程小区
3号楼302室，三人一起将杨某1控制并用黄色胶带捆绑杨某1，抢走该房内的现金人民币19000元、步步高
手机1台等物。经霞浦县价格认证中心鉴定，该部手机的价值为人民币1271元。
```
- 案例长度: 189字符
- 案例类型: 抢劫案，包含多人物、时间、地点、金额等要素

---

## 测试结果

### 1. BERT方案（tokenizer + 规则匹配）

#### 性能指标
- **分词耗时**: 0.0024秒
- **实体识别耗时**: <0.0001秒（几乎瞬时）
- **总耗时**: ~0.0024秒

#### 抽取结果
识别到5种实体类型：
- **人物**: 被告人
- **时间**: 2016年5月29日, 19时
- **地点**: 霞浦县, 牙城镇
- **金额**: 19000元, 1271元
- **行为**: 抢走, 控制, 捆绑

#### 优点
✓ 速度极快（毫秒级）  
✓ 可以做简单的关键词匹配  
✓ 资源占用少  
✓ 不需要网络调用  

#### 缺点
✗ 无法理解语义  
✗ 无法抽取复杂关系  
✗ 需要预定义规则  
✗ 无法识别实体之间的关联  
✗ 只能做表面匹配，容易误判  

---

### 2. Ollama方案（qwen25chat）

#### 性能指标
- **总耗时**: 34.29秒
- **速度**: 比BERT慢约 **14,466倍**

#### 抽取结果
- **总节点数**: 9个
- **总关系数**: 15条
- **节点类型分布**:
  - 人物: 3个
  - 案件: 2个
  - 物品: 2个
  - 受害人: 1个
  - 地点: 1个

#### 示例实体（前5个）
1. [案件] 案件 test-001: 当前输入案情的根节点
2. [案件] ××2016年5月29日的抢劫案: 一起发生在2016年5月29日晚上的抢劫案
3. [人物] 杨1某: 被告人之一，伙同他人实施抢劫
4. [人物] 杨某2: 与杨1某一同参与抢劫的同案犯
5. [人物] 李某1: 与杨1某、杨某2一同参与抢劫的同案犯，已判刑

#### 优点
✓ 可以理解语义  
✓ 可以抽取复杂实体和关系  
✓ 不需要预定义规则  
✓ 能识别实体之间的关联  
✓ 抽取结果更丰富、更准确  

#### 缺点
✗ 速度较慢（秒级）  
✗ 需要调用本地LLM  
✗ 资源占用大  
✗ 不适合大规模批量处理  

---

## 对比总结

| 维度 | BERT | Ollama | 差距 |
|------|------|--------|------|
| **速度** | 0.0024秒 | 34.29秒 | **14,466x** |
| **实体数量** | 5类关键词 | 9个结构化节点 | Ollama更丰富 |
| **关系抽取** | ❌ 不支持 | ✅ 15条关系 | Ollama独有 |
| **语义理解** | ❌ 无 | ✅ 有 | Ollama独有 |
| **准确性** | 低（规则匹配） | 高（语义理解） | Ollama更准确 |
| **可扩展性** | 需要人工规则 | 自动理解 | Ollama更灵活 |

---

## 结论与建议

### 如果用BERT替代Ollama做实体抽取：

#### 优势
✓ **速度提升约14,000倍**（从34秒降到0.002秒）  
✓ 适合大规模批量处理  
✓ 资源占用少  

#### 劣势
✗ **抽取质量大幅下降**  
✗ 无法抽取复杂的实体关系  
✗ 需要大量人工规则维护  
✗ 无法理解语义，容易误判  
✗ 难以处理复杂案例  

### 推荐方案

#### 方案1: 混合方案（推荐）
1. **第一阶段**: 用BERT做快速初步筛选
   - 识别基本实体（人物、时间、地点、金额）
   - 过滤掉明显不相关的内容
   - 耗时: 毫秒级

2. **第二阶段**: 对重要案例用Ollama做精细抽取
   - 抽取复杂实体和关系
   - 生成知识图谱
   - 耗时: 秒级

**优势**: 兼顾速度和质量

#### 方案2: 训练专门的BERT实体抽取模型
- 需要标注大量训练数据
- 训练一个BERT-NER模型
- 可以达到较好的速度和质量平衡
- **但需要大量前期投入**

#### 方案3: 保持现状
- 继续使用Ollama做实体抽取
- 接受较慢的速度
- 保证抽取质量
- **适合对质量要求高的场景**

## 针对你的四智能体系统的建议

你的系统有4个智能体：
1. **FactExtractorAgent** - 实体抽取（当前用Ollama）
2. **ChargePredictorAgent** - 罪名预测（当前用BERT）
3. **LawRetrieverAgent** - 法条检索
4. **ReportGeneratorAgent** - 报告生成（当前用Ollama）

### 建议：不要完全替换，而是分层处理

```
输入案情文本
    ↓
[BERT快速预处理] ← 新增
- 分词
- 基本实体识别
- 关键信息提取
    ↓
[Ollama精细抽取] ← 保留
- 复杂实体抽取
- 关系抽取
- 知识图谱构建
    ↓
后续智能体处理
```

### 具体实施
1. **保留Ollama的实体抽取能力**（质量优先）
2. **在前面加一层BERT预处理**（速度优化）
3. **BERT负责**:
   - 文本分词
   - 基本实体标记
   - 为Ollama提供结构化输入
4. **Ollama负责**:
   - 深度语义理解
   - 复杂关系抽取
   - 知识图谱生成

### 预期效果
- 整体速度提升: 20-30%
- 抽取质量: 保持不变
- 系统复杂度: 略微增加
